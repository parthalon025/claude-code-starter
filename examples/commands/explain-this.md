Explain this project (or the specified file/folder) as if the reader has never seen it before.

Cover:
1. **What it does** -- one sentence, plain English
2. **How it's organized** -- folder structure, key files
3. **How it works** -- high-level flow (request comes in, goes through X, returns Y)
4. **Key concepts** -- any domain-specific terms or patterns used
5. **How to run it** -- commands to start, test, build

If a specific file is given, explain that file line by line. Define technical terms inline. Use analogies when helpful.
